## Color

From `frontend/src/app.css` tokens and Bootstrap 5.3 semantic colors:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8f9fa` (`--color-surface`) | Page background, table backgrounds (`table-light`) |
| Secondary (30%) | `#ffffff` + `#dee2e6` border | Cards, table rows, modal backgrounds |

Accent reserved for:
1. "Main" advancement badge on group standings rows (COMBINED tournaments only)

- `warning` with `text="dark"` (`#ffc107`): Tiebreaker criterion badge when manual override applied (existing pattern, Phase 29)
- `success` (`#198754`): Winner highlighting in bracket view (existing)

## Component Inventory

### New components for this phase

**AdvancementBadge** (inline, within GroupStandingsTable row)
- React Bootstrap `<Badge>` component
- Main bracket: `bg="primary"` — label "Main"
- Secondary bracket: `bg="secondary"` — label "Secondary"
- Style: `fontSize: '10px'` (component-local override — see Typography section), `ms-1` (4px left margin from player name)
- Placement: immediately after player name text in the standings `<td>`, same cell
- Appears only when group matches are all complete (all group matches in terminal state)
- Non-advancing players: no badge rendered (absence is the signal — D-13)
- Same badge for projected and confirmed advancement (D-15)

Source: D-12, D-13, D-15 from CONTEXT.md; badge pattern from existing `tiebreakerCriterion` Badge at GroupStandingsTable.jsx line 180-188.

### Modified components

**GroupStandingsTable.jsx**
- Accept new prop: `advancementMap` — `Object.<entityId, 'MAIN'|'SECONDARY'|null>` — derived by parent from advancement config + current standings
- Render AdvancementBadge inline after entity name when `advancementMap[stats.entity.id]` is truthy
- No layout changes; badge fits within existing player name `<td>`

**TournamentPointConfigPage.jsx** (or equivalent organizer flow)
- "Calculate Points" button: disabled with tooltip when unresolved ties exist
- Disabled state copy: "Resolve all tied positions before calculating points"
- On GROUP-only tournament: button visible (same UX as knockout — D-10)
- Blocked state visual hierarchy: the Bootstrap `danger` alert is the primary focal point (high-contrast red, full-width, descriptive message); the disabled button + tooltip is the secondary affordance (button grayed out, tooltip appears on hover/focus)
- Error alert (Bootstrap `danger` variant): "Cannot calculate points — [N] group(s) have unresolved tied positions. Use the standings tiebreaker resolution tool first."

**FormatVisualization.jsx**
- GROUP-only path: already implemented (D-09). No UI changes required. Confirmed by CONTEXT.md D-09 and codebase note.

## States and Interactions

### Advancement Badge Display

| Condition | Badge Shown |
|-----------|-------------|
| Group has unfinished matches | No badge on any row |
| Group all matches complete, player projected to main bracket | "Main" badge (primary) |
| Group all matches complete, player projected to secondary bracket | "Secondary" badge (secondary) |
| Group all matches complete, player not advancing | No badge |
| Tournament format is GROUP-only | No badge (no advancement exists) |

Projection logic (D-14): derived client-side from `standings.position` + `formatConfig.advancePerGroup` + `tournament.mainBracketSize`/`secondaryBracketSize`. If no advancement config is set, no badges shown.

### Calculate Points — Blocked State

- Button `disabled` attribute set
- `OverlayTrigger` with `Tooltip`: "Resolve all group tiebreakers first"
- Alert `variant="danger"` below button with specific message listing which groups have unresolved ties
- Error text pattern: "Groups [Group A, Group B] have unresolved tied positions. Open each group's standings and use 'Resolve Tie' before calculating points."
- Visual anchor: the `danger` alert is the primary focal point; the disabled button + tooltip is secondary

### Calculate Points — Success State

- Existing SWR invalidation pattern after successful API call (D-10 trigger is manual, same as knockout)
- Alert `variant="success"` with text: "Points calculated and awarded successfully."
