**Step 1: Fix LanguageSwitcher.jsx** — Convert from raw Bootstrap `data-bs-toggle="dropdown"` to React Bootstrap `Dropdown` component. Import `Dropdown` from `react-bootstrap`. Replace the raw HTML dropdown with `Dropdown`, `Dropdown.Toggle`, `Dropdown.Menu`, `Dropdown.Item`. Keep the same visual styling (link-style toggle, flag + code display). This is a small change — the component interface (no props, default export) stays identical.

**Step 2: Refactor NavBar.jsx** — Complete replacement of raw HTML Bootstrap markup with React Bootstrap components. Follow the 21-RESEARCH.md architecture patterns exactly.

Core structure:
1. Add `const [show, setShow] = useState(false)` for controlled Offcanvas state
2. Add `handleClose = () => setShow(false)` and `handleToggle = () => setShow(prev => !prev)`
3. Add `handleNav = (path) => navigate(path)` helper (collapseOnSelect handles close via eventKey)
4. Replace `<nav className="navbar...">` with `<Navbar expand="lg" collapseOnSelect expanded={show} onToggle={handleToggle} variant="dark" bg="primary">`
5. Replace the `data-bs-toggle="collapse"` button with `<Navbar.Toggle aria-controls="batl-offcanvas" />`
6. Replace `<div className="collapse navbar-collapse">` with `<Navbar.Offcanvas id="batl-offcanvas" placement="end" show={show} onHide={handleClose} scroll={false} backdrop={true}>`
7. Add `<Offcanvas.Header closeButton><Offcanvas.Title>Menu</Offcanvas.Title></Offcanvas.Header>`
8. Wrap nav content in `<Offcanvas.Body className="d-flex flex-column">`

Nav link conversion — convert ALL `<span className="nav-link" onClick={() => navigate(path)}>` to `<Nav.Link eventKey="unique-key" onClick={() => handleNav(path)}>`. Every Nav.Link MUST have a unique eventKey or collapseOnSelect will silently fail. Remove all `<li className="nav-item">` wrappers and `<ul className="navbar-nav">` — use `<Nav>` instead.

Link grouping in the drawer:
- **Public links** (no section header): Rankings, Tournaments (for visitors + PLAYER)
- **Organizer section** (for ORGANIZER role only, NOT ADMIN): Muted uppercase section header "Organizer" + `<hr>` divider, then Categories, Tournaments, Players, Doubles Pairs links
- **Admin section** (for ADMIN role only): Use React Bootstrap `<Accordion flush>` with 3 `<Accordion.Item>` sections. IMPORTANT: Place the Accordion OUTSIDE the `<Nav>` element (as a sibling) to prevent Accordion.Header clicks from triggering collapseOnSelect and closing the drawer. The Accordion items are:
  - "Organizer" (eventKey="admin-org"): Categories, Tournaments, Players, Doubles Pairs
  - "Admin" (eventKey="admin-admin"): Users, Point Tables
  - "Player Access" (eventKey="admin-player"): Registrations, Categories
  Nav.Link elements inside Accordion.Body MUST still have unique eventKeys prefixed with "adm-" to avoid collision.
- **Player section** (for PLAYER role only): Muted section header "Player" + divider, then Registrations, Categories, Doubles Pairs

Drawer footer (mt-auto, anchored at bottom):
- `<div className="mt-auto border-top pt-3 pb-2">`
- `<LanguageSwitcher />` — rendered inside drawer on mobile
- Logged-in state: role badge (`<span className="badge bg-primary">`) + email (text-truncate) + logout button
- Logged-out state: Login (btn-outline-primary) and Register (btn-primary) buttons side by side

Desktop behavior preservation:

- On desktop, LanguageSwitcher and user info should display in the navbar just like they do now — the Offcanvas.Body content renders inline at lg+
- Keep `handleHomeClick` logic for brand click (role-based navigation)

Important anti-patterns to AVOID (from research):
- Do NOT use `data-bs-toggle` or `data-bs-target` anywhere — these require Bootstrap JS
- Do NOT use `<span>` elements for navigation — they bypass React Bootstrap's Nav event system
- Do NOT put Nav.Link inside Accordion.Body without unique eventKeys
- Do NOT nest Accordion inside Nav — put them as siblings inside Offcanvas.Body
- Do NOT use `href` on Nav.Link — it causes full page reload; use `eventKey` + `onClick`
