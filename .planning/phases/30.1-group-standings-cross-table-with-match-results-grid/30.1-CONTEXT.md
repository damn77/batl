# Phase 30.1: Group Standings Cross-Table with Match Results Grid - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a cross-table (matrix) showing head-to-head match results to the GroupStandingsTable component, with cell-click result entry, tabbed navigation between cross-table/standings/matches, and bidirectional cross-highlighting between cross-table cells and match list rows. Covers cross-table layout, cell interaction, responsive tab structure, and cross-highlighting. New tiebreaker logic, advancement changes, and match list reordering are out of scope.

</domain>

<decisions>
## Implementation Decisions

### Tab structure (responsive)
- **Desktop:** Two tabs — "Results" (cross-table, default) and "Standings" (existing stats table). Match list always visible below the active tab
- **Mobile (375px):** Three tabs — "Results", "Standings", "Matches". Match list moves into its own tab to save vertical space
- Cross-table is the default/active tab on both breakpoints — it's the primary view

### Cross-table layout
- **Column headers:** Player initials (first letters of first+last name, e.g., "AL" for Alice). Row labels also use initials with full name in the Player column
- **Cell content (multi-set matches):** Set score from row player's perspective (e.g., "2-0", "1-2")
- **Cell content (single set/tiebreak):** Show the full game score instead (e.g., "6-4", "7-5") since set score would just be "1-0" which is meaningless
- **Unplayed cells:** Empty / dash
- **Diagonal cells:** Greyed out, non-interactive (no cursor change, no click handler)
- **Win/loss styling:** Bold green text for wins, muted/grey text for losses (from row player's perspective). No background color fills on cells

### Cell interaction
- **All match cells are clickable** — empty cells open MatchResultModal for entry, completed cells open for viewing/editing (if authorized)
- **Row player = Player 1** in the modal — clicking row Alice, column Bob opens modal with Alice as Player 1
- **Symmetric update:** After result submission, both cells [A,B] and [B,A] update immediately (via SWR mutation)
- Reuses existing `MatchResultModal` component — no new score entry UI

### Cross-highlighting (desktop only)
- **Bidirectional:** Hovering a match row in the match list highlights both corresponding cross-table cells (background highlight color, e.g., light blue). Hovering a cross-table cell highlights the corresponding match row in the list
- **Both symmetric cells highlight** — hovering cell [AL,BO] highlights both [AL,BO] and [BO,AL] plus the match row
- **Mobile:** Cross-highlighting is skipped entirely. The three-tab layout separates concerns enough that visual linking isn't needed on mobile

### Match list
- Match list remains as-is from Phase 28 (clickable rows opening MatchResultModal)
- On desktop: always visible below the tab content (Results or Standings)
- On mobile: its own "Matches" tab

### Claude's Discretion
- Mobile cross-table horizontal scroll behavior (if table exceeds viewport)
- Exact highlight color for cross-highlighting
- Tab component choice (React Bootstrap Nav/Tabs)
- Doubles initials format (pair initials vs combined)
- Match number display in match list rows
- Exact responsive breakpoint for 2-tab vs 3-tab switch

</decisions>

<specifics>
## Specific Ideas

- Cross-table is the "hero" view — more important than standings for day-to-day tournament monitoring
- Single set/tiebreak matches show full game score (6-4) instead of meaningless "1-0" set score
- Desktop always shows match list below tabs so users don't need to switch contexts to see upcoming matches
- Cross-highlighting helps users quickly find "when does this matchup get played?" by hovering a cell and seeing it light up in the match list

</specifics>
