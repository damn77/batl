Add a new exported function `submitMatchResultDryRun` that calls the same endpoint with `?dryRun=true`:

```javascript
/**
 * Dry-run a match result submission to detect downstream impact.
 * Returns impact data without modifying any match records.
 *
 * @param {string} matchId - Match UUID
 * @param {Object} resultData - Result payload (same as submitMatchResult)
 * @returns {Promise<Object>} Impact summary: { dryRun, impactedMainMatches, impactedConsolationMatches, totalImpactedMatches, affectedPlayers, requiresConfirmation }
 */
export async function submitMatchResultDryRun(matchId, resultData) {
  const response = await apiClient.patch(`/v1/matches/${matchId}/result?dryRun=true`, resultData);
  return response.data.data;
}
```

**A. Track existing winner for change detection:**
Add a derived value from the parsed matchResult:
```javascript
const existingWinner = matchResult?.winner || null;
```

This tells us if the match already has a result (resubmission) and what the original winner was.

**B. Winner-change detection:**
Add a computed boolean:
```javascript
const isWinnerChanging = existingWinner && formValue.winner && formValue.winner !== existingWinner;
```

**C. Non-organizer winner lock (BB-01):**
For non-organizer mode (Mode 2: Player editable), the SetsScoreForm and BigTiebreakForm already drive the winner from scores. The key change: if the form's winner differs from existingWinner AND the user is NOT an organizer, show an inline Alert (variant="info") below the form:

```
"Only a tournament organizer can change the winner of a completed match. You can update the score without changing the winner."
```

Disable the submit button when `!isOrganizer && isWinnerChanging` is true. This prevents the API call from happening. The backend 403 serves as defense in depth.

For the organizer's "Enter score" mode, winner radio buttons (inside SetsScoreForm/BigTiebreakForm) remain fully editable. For non-organizers, the winner is derived from scores (the score form auto-determines winner), so if they enter scores that flip the winner, the alert appears and submit is blocked.

**D. Score-only shortcut (BB-02):**
In `handleSubmit`, after building `resultData`, check:
- If `existingWinner` is set AND `resultData.winner === existingWinner` (same winner), call `submitMatchResult` directly — NO dry-run, no confirmation. This is a score-only correction.

**E. Organizer winner-change flow with dry-run (BB-04):**
In `handleSubmit`, after building `resultData`, check:
- If `isOrganizer` AND `existingWinner` is set AND `resultData.winner !== existingWinner` (winner changing):
  1. Call `submitMatchResultDryRun(match.id, resultData)`.
  2. If response `requiresConfirmation` is true (totalImpactedMatches > 0):
     - Set a new state `pendingWinnerChange` with the dry-run response data + resultData:
       ```javascript
       setPendingWinnerChange({
         resultData,
         impact: dryRunResult
       });
       ```
     - Return early (don't submit yet).
  3. If `requiresConfirmation` is false (0 impacted matches):
     - Proceed directly with `submitMatchResult(match.id, resultData)` — no popup needed.

**F. New state for winner-change confirmation:**
Add state:
```javascript
const [pendingWinnerChange, setPendingWinnerChange] = useState(null);
```

Reset it in the existing match-change useEffect (alongside pendingInvalidSubmit reset):
```javascript
setPendingWinnerChange(null);
```

**G. Confirmation popup UI:**
Reuse the `pendingInvalidSubmit` pattern. When `pendingWinnerChange` is set, render a warning Alert (variant="warning") in Modal.Body:

```jsx
{pendingWinnerChange && (
```


**I. handleConfirmWinnerChange:**
```javascript
const handleConfirmWinnerChange = async () => {
  if (!pendingWinnerChange) return;
  setSubmitting(true);
  try {
    await submitMatchResult(match.id, pendingWinnerChange.resultData);
    setPendingWinnerChange(null);
    mutate();
    if (match?.tournamentId) {
      globalMutate(key => typeof key === 'string' && key.includes(`/tournaments/${match.tournamentId}/matches`));
    }
    onClose();
  } catch (err) {
    setError(err.message || t('errors.genericFallback', 'An error occurred'));
  } finally {
    setSubmitting(false);
  }
};
```
