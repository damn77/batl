- Added `submitMatchResultDryRun` to matchService.js — calls `?dryRun=true` endpoint, returns impact summary

- Non-organizer winner-lock: inline info Alert + disabled submit button when form winner differs from stored winner
- Organizer score-mode: dry-run triggered when `isWinnerChanging`, confirmation popup if `requiresConfirmation`, direct submit if 0 impacted
- Organizer special-outcome mode: `isSpecialWinnerChanging` (from `specialWinner` state) triggers same dry-run flow correctly
- Score-only corrections (same winner) short-circuit before dry-run — no extra round trip
- Confirmation popup shows impacted main/consolation match counts and affected player names
- `handleConfirmWinnerChange` executes actual submission and refreshes both bracket SWR caches
