Create a new component `frontend/src/components/ManualDrawEditor.jsx` that provides the manual draw assignment grid.

**Props:**
```javascript
ManualDrawEditor.propTypes = {
  tournament: PropTypes.object.isRequired,        // full tournament object
  matches: PropTypes.array.isRequired,            // all matches from useMatches
  rounds: PropTypes.array.isRequired,             // rounds from structure.rounds
  registeredPlayers: PropTypes.array.isRequired,  // from GET /tournaments/:id/registrations
  isDoubles: PropTypes.bool.isRequired,
  onAssignmentChange: PropTypes.func,             // callback after successful assign/clear to trigger SWR mutate
};
```

**Component Implementation:**

1. **Compute placed entities from matches.**
   Filter matches to Round 1 only (using `rounds` to find the round with `roundNumber === 1` for the MAIN bracket). For each Round 1 match, collect the placed player/pair IDs (from `player1Id`/`player2Id` for singles, `pair1Id`/`pair2Id` for doubles). Build a `Set` of placed entity IDs. Also count: `totalSlots` = number of non-BYE Round 1 matches * 2, `filledSlots` = count of non-null entity slots in non-BYE Round 1 matches.

2. **Compute unplaced entities for dropdown options.**
   From `registeredPlayers`, filter out any whose entity ID is already in the placed set. For singles: entity ID is `reg.player.id`. For doubles: entity ID is `reg.pair.id`.

   Sort unplaced entities for the dropdown. For doubles, sort by `pair.seedingScore` descending (highest first). For singles, sort alphabetically by name (since seedingScore is not available in the registrations response -- per Claude's discretion, alphabetical is reasonable when seeding data is unavailable in the response).

3. **Progress bar.**
   Above the grid, render a Bootstrap `ProgressBar` with text:
   ```jsx
   <div className="mb-3">
     <div className="d-flex justify-content-between align-items-center mb-1">
       <span className="fw-semibold">Placement Progress</span>
       <span className={filledSlots === totalSlots ? 'text-success fw-bold' : 'text-muted'}>
         {filledSlots}/{totalSlots} positions filled
       </span>
     </div>
     <ProgressBar
       now={totalSlots > 0 ? (filledSlots / totalSlots) * 100 : 0}
       variant={filledSlots === totalSlots ? 'success' : 'primary'}
     />
   </div>
   ```

4. **Assignment grid.**
   Group Round 1 matches by round (there is only one Round 1, but use the same pattern as existing code). Render in a responsive `Row xs={1} md={2} lg={3}` grid.

   For each Round 1 match, render a Card:
   - **BYE matches** (`match.isBye || match.status === 'BYE'`): Show as a disabled card with `className="border h-100 bg-light opacity-75"`. Display "Match {matchNumber}" header with a BYE badge. Show both slots as greyed-out text: slot with player name shows the player name in muted text, empty slot shows "BYE" in muted italic. No dropdowns, no clear buttons.

   - **Non-BYE matches**: Show "Match {matchNumber}" header. For each slot (`player1`/`player2`):
     - **If slot has an entity assigned** (check `match.player1Id`/`match.player2Id` for singles, `match.pair1Id`/`match.pair2Id` for doubles):
       Display the entity name with a small "x" clear button next to it:
       ```jsx
       <div className="d-flex align-items-center justify-content-between">
         <span className="fw-medium">{entityName}</span>
         <Button
           variant="outline-danger"
           size="sm"
           className="py-0 px-1"
           onClick={() => handleClearSlot(match.id, slotName)}
           disabled={assigning}
           title="Clear position"
         >
           &times;
         </Button>
       </div>
       ```
     - **If slot is empty**:
       Show a `Form.Select` dropdown:
       ```jsx
       <Form.Select
         size="sm"
         value=""
         onChange={(e) => handleAssignSlot(match.id, slotName, e.target.value)}
         disabled={assigning}
       >
         <option value="" disabled>Select player...</option>
         {unplacedEntities.map(entity => (
           <option key={entity.id} value={entity.id}>
             {entity.displayName}
           </option>
         ))}
       </Form.Select>
       ```
       Where `entity.displayName` is:
       - Singles: `player.name`
       - Doubles: `player1.name + " & " + player2.name` with optional seedingScore prefix

5. **handleAssignSlot function.**
   Uses `useToast()` for error display. Uses `assignPosition` from bracketPersistenceService.
   ```javascript
   const handleAssignSlot = async (matchId, slot, entityId) => {
     setAssigning(true);
     try {
       const body = { matchId, slot };
       if (isDoubles) { body.pairId = entityId; } else { body.playerId = entityId; }
       await assignPosition(tournament.id, body);
       if (onAssignmentChange) await onAssignmentChange();
     } catch (err) {
       showError(err.message || 'Failed to assign position');
     } finally {
       setAssigning(false);
     }
   };
   ```

6. **handleClearSlot function.**
   ```javascript
   const handleClearSlot = async (matchId, slot) => {
     setAssigning(true);
     try {
       const body = { matchId, slot, playerId: null, pairId: null };
       await assignPosition(tournament.id, body);
       if (onAssignmentChange) await onAssignmentChange();
     } catch (err) {
       showError(err.message || 'Failed to clear position');
     } finally {
       setAssigning(false);
     }
   };
   ```

7. **State:** `const [assigning, setAssigning] = useState(false);` -- disables all dropdowns/buttons during API call to prevent race conditions.
