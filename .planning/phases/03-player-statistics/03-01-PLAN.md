  <name>Task 1: Add getPlayerMatchHistory() to playerService.js</name>
  <files>backend/src/services/playerService.js</files>
  <action>
Add a new named export `getPlayerMatchHistory({ playerId, page = 1, limit = 20, categoryId = null })` to the bottom of the existing playerService.js file. Do not modify existing functions.

Implementation steps:

1. Verify player exists using `prisma.playerProfile.findUnique({ where: { id: playerId }, select: { id: true } })`. If not found, return null (controller will 404).

2. Build the `where` clause:
```javascript
const where = {
  OR: [{ player1Id: playerId }, { player2Id: playerId }],
  isBye: false,
  NOT: { status: 'CANCELLED' }
};
if (categoryId) {
  where.tournament = { categoryId };
}
```

3. Count total with `prisma.match.count({ where })`.

4. Query matches with `prisma.match.findMany`:
   - Same `where` clause
   - `include`: player1 and player2 `{ select: { id, name } }`, pair1 and pair2 `{ include: { player1: { select: { id, name } }, player2: { select: { id, name } } } }`, tournament `{ select: { id, name, endDate, category: { select: { id, name } } } }`
   - `orderBy`: `[{ completedAt: 'desc' }, { createdAt: 'desc' }]` — pending matches (null completedAt) fall to end via nulls-last default in PostgreSQL
   - `skip: (page - 1) * limit`, `take: limit`

5. Map each match to a display row:
```javascript
const isPlayer1 = match.player1Id === playerId;
const opponent = isPlayer1 ? match.player2 : match.player1;
const opponentPair = isPlayer1 ? match.pair2 : match.pair1;
const opponentName = opponentPair
  ? `${opponentPair.player1.name} / ${opponentPair.player2.name}`
  : (opponent?.name || 'TBD');
```

6. Score formatting helper `formatMatchScore(result, isPlayer1)`:
   - If result is null → return `'-'`
   - Parse JSON: `const parsed = JSON.parse(result)`
   - If `parsed.outcome` → return `{ WALKOVER: 'Walkover', FORFEIT: 'Forfeit', NO_SHOW: 'No-show' }[parsed.outcome]`
   - Else if `parsed.sets` → map sets to strings from player's perspective:
     - If isPlayer1: `${set.player1Score}-${set.player2Score}`
     - Else: `${set.player2Score}-${set.player1Score}`
     - Add tiebreak if present: append `(${set.tiebreakScore})`
     - Join with `', '`
   - Return `'-'` as fallback

7. Outcome helper `determineMatchOutcome(result, isPlayer1)`:
   - If result is null → return null
   - Parse JSON
   - `const won = (parsed.winner === 'PLAYER1') === isPlayer1`
   - Return `won ? 'W' : 'L'`

8. Return row object:
```javascript
{
  matchId: match.id,
  tournamentId: match.tournament.id,
  tournamentName: match.tournament.name,
  category: match.tournament.category,   // { id, name }
  opponentName,
  score,
  outcome,     // 'W', 'L', or null
  completedAt: match.completedAt
}
```

9. Return final structure:
```javascript
return {
  matches: rows,
  pagination: { total, page, limit, totalPages: Math.ceil(total / limit) }
};
```

Use ES module syntax (`import`/`export`) consistent with the rest of the file. Add `getPlayerMatchHistory` to the default export object at the bottom of the file if one exists, otherwise only add the named export.

Add a new named export `getMatchHistoryHandler` after the existing handlers:

```javascript
export const getMatchHistoryHandler = async (req, res, next) => {
  try {
    const { id } = req.params;
    const page = parseInt(req.query.page) || 1;
    const limit = Math.min(parseInt(req.query.limit) || 20, 100); // cap at 100
    const categoryId = req.query.categoryId || null;

    const result = await getPlayerMatchHistory({ playerId: id, page, limit, categoryId });

    if (!result) {
      return res.status(404).json({
        success: false,
        error: { code: 'NOT_FOUND', message: 'Player not found' }
      });
    }

    return res.status(200).json({ success: true, data: result });
  } catch (err) {
    next(err);
  }
};
```
