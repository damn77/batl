In MatchResultModal.jsx:

1. Import ButtonGroup from react-bootstrap (add to existing import).

2. Replace the current mode toggle div (lines ~373-392, the div.mb-3 with two Form.Check inline radios) with TWO renderings controlled by CSS visibility:

   a. Desktop (>=576px): Keep the existing inline Form.Check radios wrapped in a div with className "mode-toggle-desktop d-none d-sm-block mb-3".

   b. Mobile (<576px): Add a ButtonGroup wrapped in a div with className "mode-toggle-mobile d-sm-none mb-3". The ButtonGroup should be full-width (className="w-100"). Use two Button elements:
      - "Enter score": variant={mode === 'score' ? 'primary' : 'outline-primary'}, onClick={() => setMode('score')
      - "Special outcome": variant={mode === 'special' ? 'primary' : 'outline-primary'}, onClick={() => setMode('special')
      Each button should have className="flex-fill" to share width equally.

3. In the Modal.Footer, wrap the confirmation button groups (both pendingWinnerChange and pendingInvalidSubmit branches) in a container div with className "confirmation-buttons" so CSS can target them for mobile stacking.

   For the pendingWinnerChange branch (currently Cancel + Go Back + Confirm Change):
   - Wrap in: div.confirmation-buttons.d-flex.flex-wrap.gap-2.w-100
   - Order: Confirm Change button first (variant="danger", className="confirm-primary"), then Go Back (variant="outline-secondary", className="confirm-secondary"), then Cancel (variant="secondary", className="confirm-secondary")
   - Remove the standalone Cancel button from Modal.Footer when in confirmation mode — integrate it into the confirmation-buttons div.

   For the pendingInvalidSubmit branch (currently Cancel + Fix Scores + Submit Anyway):
   - Same wrapper: div.confirmation-buttons.d-flex.flex-wrap.gap-2.w-100
   - Order: Submit Anyway first (variant="danger", className="confirm-primary"), then Fix Scores (variant="outline-secondary", className="confirm-secondary"), then Cancel (variant="secondary", className="confirm-secondary")
   - Same Cancel integration as above.

   For the default (non-confirmation) footer: keep Cancel + Save Result as-is — no changes needed.

4. In MatchResultModal.css, add within the existing @media (max-width: 575.98px) block:

   ```css
   /* Confirmation buttons: primary full-width on top, secondaries side-by-side below */
   .match-result-modal .confirmation-buttons {
     flex-direction: column;
   }

   .match-result-modal .confirmation-buttons .confirm-primary {
     width: 100%;
     order: -1;
   }

   .match-result-modal .confirmation-buttons .confirm-secondary {
     flex: 1;
   }
   ```

   This makes the primary action (Confirm Change / Submit Anyway) full-width on the first row, and the two secondary actions (Go Back + Cancel, or Fix Scores + Cancel) share the second row side-by-side.

5. Also add a rule for the segmented ButtonGroup tap target height:
   ```css
   .match-result-modal .mode-toggle-mobile .btn {
     min-height: 44px;
     font-size: 1rem;
   }
   ```
