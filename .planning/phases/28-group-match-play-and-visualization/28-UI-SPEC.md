## Spacing Scale

Declared values from `frontend/src/App.css` — project-established tokens:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xs: 0.25rem`) | Icon gaps, badge padding, tight inline elements |
| sm | 8px (`--spacing-sm: 0.5rem`) | Card body padding (mobile), compact element gaps |
| md | 12px (`--spacing-md: 0.75rem`) | Card body padding (desktop), section spacing within groups |
| lg | 16px (`--spacing-lg: 1rem`) | Default element and column spacing |
| xl | 24px (`--spacing-xl: 1.5rem`) | Section separation (group stage vs knockout sections) |
| 2xl | 32px | Major section breaks (not tokenized; use Bootstrap `mb-4`) |
| 3xl | 48px | Page-level spacing (not tokenized; use Bootstrap `py-5`) |

Exceptions:
- Match row tap target: 44px minimum height on mobile (< 576px) — enforced globally by `App.css` `.list-group-item-action` and `btn` rules. Apply `style={{ minHeight: '44px' }}` directly on clickable `<tr>` elements since table rows are not covered by the global rule.
- Progress bar height: 4px (thin indicator under accordion header — not from spacing scale; hardcoded `style={{ height: '4px' }}`).
- Accordion header progress bar margin-top: 6px (visual breathing room between group name and bar — not from spacing scale; hardcoded).

Source: `frontend/src/App.css`, `CONTEXT.md` Mobile responsiveness decisions, `RESEARCH.md` Pattern 1 and Pattern 4

## Typography

Project-established tokens from `frontend/src/App.css`:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px (`--font-size-base: 0.9375rem`) | 400 (regular) | 1.5 (Bootstrap default) |
| Label / small / table cell | 13px (`--font-size-sm: 0.8125rem`) | 400 (regular) | 1.4 |
| Section heading (group name, h6) | 17px (`--font-size-lg: 1.0625rem`) | 600 (semibold, via `fw-bold`) | 1.2 |
| Legend text (`small.text-muted`) | 13px | 400 | 1.4 |

Typography rules for this phase:
- Standings position column: `fw-bold` at 15px
- Standings player name: `fw-bold` at 15px (existing pattern)
- Standings points column: `fw-bold` at 15px (existing pattern)
- Differential values (S +/-, G +/-): 15px weight 400; positive prefix `+` rendered explicitly (e.g., `+3`); zero rendered as `0`; negative rendered as `-2`
- Doubles pair display: `"Player A / Player B"` — 15px weight 400, single line (truncate with `text-truncate` if needed)
- Group completion banner heading: `fw-bold` at 15px inline within `<Alert>`
- "Knockout bracket will appear after group stage" placeholder: 15px weight 400 inside `<Alert variant="info">`
- Match row player names on mobile: 13px to fit compact single-row format

Source: `frontend/src/App.css` design tokens, `CONTEXT.md` standings columns decision, `RESEARCH.md` Pitfall 6

## Color

Project uses Bootstrap 5.3 semantic colors. No custom hex values — use Bootstrap variant names.

| Role | Bootstrap Value | Usage |
|------|-----------------|-------|
| Dominant (60%) | `#f8f9fa` (`--color-surface`, Bootstrap `bg-light`) | Page background, table header (`table-light`), card headers, accordion body background |
| Secondary (30%) | `#ffffff` (Bootstrap default card/modal white) | Card bodies, modal body, standings table rows, accordion items |
| Accent (10%) | Bootstrap `primary` (`#0d6efd` default) | Primary CTA button "Generate Knockout Bracket", progress bar fill (`variant="success"`), active accordion item indicator |
| Destructive | Bootstrap `danger` (`#dc3545`) | Match load error Alert only; no destructive actions in this phase |

Accent reserved for:
1. "Generate Knockout Bracket" action button (`variant="primary"`) in group completion banner
2. Progress bar fill indicating group completion (`<ProgressBar variant="success" />`) — uses success, not primary, to signal positive completion state
3. Active/expanded Accordion.Item — Bootstrap renders the header button in primary by default; leave default styling

Status badge colors (existing `MatchResultDisplay` pattern — do not change):
- SCHEDULED: `variant="secondary"` (grey)
- IN_PROGRESS: `variant="primary"` (blue)
- COMPLETED: `variant="success"` (green)
- CANCELLED: `variant="danger"` (red)

Winner highlighting in match rows: `text-success` (`fw-bold text-success`) on winning player name — existing `MatchResultDisplay` pattern, reuse unchanged.

Group completion banner: `<Alert variant="success">` — green surface signals a positive milestone (group stage done).

"Knockout bracket will appear after group stage" placeholder: `<Alert variant="info">` — blue neutral information.

Source: `frontend/src/App.css` color tokens, `RESEARCH.md` Pattern 5, existing `MatchResultDisplay.jsx` status badge config

## Interaction Contract

### Accordion Behavior (Groups)

- Implementation: `<Accordion defaultActiveKey={tournamentStatus === 'IN_PROGRESS' ? groups[0]?.id : null}>`
- One group expanded at a time (React Bootstrap Accordion native behavior)
- IN_PROGRESS tournaments: first group auto-expanded on mount
- COMPLETED/other status tournaments: all groups collapsed on mount
- Accordion header content: group name (left) + participant count Badge (right of name) + thin ProgressBar below name text
- Chevron (Bootstrap `::after` pseudo-element) must remain visible — progress bar div uses `style={{ flexGrow: 1, marginRight: '1rem' }}` inside header to avoid overflow

### Match Row Interaction

- Each match row in the matches section is clickable: `style={{ cursor: 'pointer', minHeight: '44px' }}` on the row element
- Click handler: `setSelectedMatch(match)` to open MatchResultModal
- Row contents (single line, compact): `[Status Badge] [Player 1 name] vs [Player 2 name] [score if completed]`
- For DOUBLES: `[Status Badge] [P1A / P1B] vs [P2A / P2B] [score if completed]`
- Completed match rows reuse `MatchResultDisplay` in compact mode for score/winner display
- Scheduled match rows show status badge + player names + "vs" (no score)

### MatchResultModal Skip Dry-Run for Group Matches

- Group matches have `match.groupId` set — modal must skip `runDryRunIfNeeded` when `match.groupId` is truthy
- No confirmation modal for organizer winner corrections on group matches (CONTEXT.md decision)
- After submit: call `mutateMatches()` (group match cache) + `globalMutate` (all tournament match caches)

### Progress Bar States

- 0% (no matches completed): empty bar, `now={0}`, `variant="success"`
- Partial: fills proportionally, `now={completionPct}`, `variant="success"`
- 100% (all matches completed): full green bar
- `style={{ height: '4px', marginTop: '6px' }}` inside accordion header

### Group Completion Banner (COMBINED only)

- Shown when: `isOrganizer && groupsComplete && brackets.length === 0`
- Placement: top of `CombinedFormatDisplay`, above group stage section
- Component: `<Alert variant="success" className="d-flex justify-content-between align-items-center">`
- Left content: `<strong>Group stage complete</strong> — Ready to generate knockout bracket`
- Right content: `<Button variant="primary" size="sm" onClick={onGenerateKnockout}>Generate Knockout Bracket</Button>`
- Dismissed when: brackets are generated (banner condition becomes false)

### Knockout Bracket Placeholder (COMBINED, group stage not done)

- Shown when: `brackets.length === 0` (regardless of organizer status — visible to all)
- Component: `<Alert variant="info">Knockout bracket will appear after group stage completes.</Alert>`
- Placement: in the "Knockout Phase" section of `CombinedFormatDisplay`

### Mobile Column Hiding (< 576px)

Hidden columns at breakpoint < 576px (Bootstrap `d-none d-sm-table-cell` on both `<th>` and `<td>`):
- Sets W-L column
- Games W-L column

Always visible (all breakpoints):
- `#` (position)
- Player
- P (played)
- W (wins)
- L (losses)
- S +/- (set differential)
- G +/- (game differential)
- Pts (points)

Source: `CONTEXT.md` mobile decisions, `RESEARCH.md` Pattern 4

| Match row: completed | Show score inline via MatchResultDisplay compact mode |
