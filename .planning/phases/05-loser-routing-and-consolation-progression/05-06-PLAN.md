Backend endpoint contract (consolationOptOutController.js):
```
POST /api/v1/tournaments/:id/consolation-opt-out
Authorization: authenticated (player self-service or organizer)
Body: { playerId: string } (singles) OR { pairId: string } (doubles)
Responses:
  201 { success: true, data: ConsolationOptOut }
  400 MISSING_ENTITY — neither playerId nor pairId provided
  400 NOT_CONSOLATION_TOURNAMENT — tournament is not MATCH_2
  403 NOT_AUTHORIZED — player attempting to opt out someone else
  409 ALREADY_OPTED_OUT — duplicate opt-out attempt
  409 NEXT_MATCH_ALREADY_PLAYED — consolation match already completed
```

apiClient import pattern (used by all frontend services):
```javascript
import apiClient from '../services/apiClient';
// Usage:
const response = await apiClient.post(`/v1/tournaments/${tournamentId}/consolation-opt-out`, { playerId });
// Errors are pre-transformed: err.message = human-readable, err.code = machine code
```

Registration data shape (from getTournamentRegistrations):
```javascript
// Singles:
{ id, status, player: { id, name }, seedPosition }
// Doubles:
{ id, status, pair: { id, player1: { id, name }, player2: { id, name } }, seedPosition }
// type: 'SINGLES' | 'DOUBLES'
```

Tournament object shape (from useTournament hook):
```javascript
{
  id, name, status,
  formatType: 'KNOCKOUT',
  formatConfig: { matchGuarantee: 'MATCH_2' }  // parsed object, not a JSON string
}
```

Auth context (useAuth hook):
```javascript

// user.role: 'PLAYER' | 'ORGANIZER' | 'ADMIN'
// user.playerId: UUID of player profile (null for organizers without profile)
```
