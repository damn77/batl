## Spacing Scale

This phase uses Bootstrap spacing utilities exclusively. Token names map to Bootstrap spacing classes.

| Token | Value | Bootstrap Class | Usage |
|-------|-------|-----------------|-------|
| xs | 4px | `mb-1`, `me-1`, `py-1` | Icon gaps, tight inline separators |
| sm | 8px | `mb-2`, `me-2`, `gap-2`, `py-2` | List item padding, badge spacing |
| md | 16px | `mb-3`, `me-3`, `px-3`, `py-3` | Default form group spacing, card body |
| lg | 24px | `mb-4`, `gap-4` | Section separators within a Card |
| xl | 32px | `mb-5` | Card-to-card vertical rhythm |

Exceptions:
- Group size preview inline badge: `ms-2` (8px) after group count input — consistent with knockout Badge use
- Seed position Badge in swap selector: `me-1` (4px) before player name — tight inline labeling

## Color

Uses Bootstrap semantic color tokens. No custom hex values — all colors come from Bootstrap 5.3 CSS variables.

| Role | Bootstrap Semantic | Usage |
|------|-------------------|-------|
| Dominant (60%) | `bg-white` / `bg-light` | Card body backgrounds, page surface |
| Secondary (30%) | `bg-light` (Card.Header), `table-light` (thead) | Card headers, table headers, section dividers |
| Accent (10%) | `variant="primary"` | Generate Draw button, Close Registration button |
| Success | `variant="success"`, `bg="success"` | "Generated" status Badge, success Alert after draw completes |
| Warning | `variant="warning"`, `variant="outline-warning"` | Revert to Scheduled button only |
| Destructive | `variant="danger"`, `variant="outline-danger"` | Regenerate Draw button, error Alert |
| Info | `variant="info"` | Unsaved-changes Alert, validation-in-progress note |

Accent (primary) reserved for: Generate Group Draw button, Close Registration button — the single forward-progress action per state. No other elements use `variant="primary"`.

## Component Inventory

All components are extensions or reuses of existing project components. No new design primitives.

### New Component: GroupDrawGenerationSection

Extends `BracketGenerationSection` pattern. Three states identical in structure to knockout:

**State A — Registration open:**
- `Card` with `Card.Header` ("Draw Generation") and `Card.Body`
- Player/pair count sentence: `<strong>{N}</strong> player{s} registered.`
- `Button variant="primary" size="lg"` labeled "Close Registration"
- Inline `Spinner` during async operation

**State B — Registration closed, no draw yet:**
- `Card.Header`: h5 "Draw Generation" + `Badge bg="success"` "Registration closed"
- Registered player/pair list: `ListGroup variant="flush"` with numbered items
  - Singles: player name only
  - Doubles: "Name1 & Name2"
- Group configuration form (see sub-section below)
- `Button variant="primary" size="lg"` labeled "Generate Group Draw"
- Minimum player warning: `<small className="text-warning">` below button when player count < 4

**State C — Draw generated:**
- `Card.Header`: h5 "Draw" + `Badge bg="success"` "Generated"
- Header action buttons (right-aligned): Regenerate Draw (`outline-danger`), Revert to Scheduled (`outline-warning`)
- Body: one `GroupStandingsTable` per group (existing component, empty standings on first render)
- Generating spinner overlay: centered `Spinner` + `<p className="text-muted mt-2">`

**State D — IN_PROGRESS (locked):**
- Same as knockout revert-only state: Card with "Revert to Scheduled" (`outline-warning`) button and explanatory text
- No draw editing when tournament is IN_PROGRESS

### Group Configuration Form (State B only)

Rendered inside `Card.Body` above the Generate button.

```
Group Count input
  <Form.Group className="mb-4">
    <Form.Label className="fw-semibold">Number of Groups</Form.Label>
    <Form.Control type="number" min="2" max={Math.floor(playerCount/2)} />
    <Form.Text className="text-muted">
      Enter the number of groups. Group sizes are calculated automatically.
    </Form.Text>
  </Form.Group>

Group Size Preview (appears as soon as groupCount is valid)
  <Alert variant="info" className="py-2">
    <small>{groupCount} groups: {previewText}</small>
  </Alert>
  — previewText example: "2 of 4 players, 2 of 3 players"
  — renders inline below the input, dismissible=false

Seeded Rounds input
  <Form.Group className="mb-4">
    <Form.Label className="fw-semibold">Seeded Rounds</Form.Label>
    <Form.Control type="number" min="0" max={groupCount} />
    <Form.Text className="text-muted">
      Number of rounds to fill using snake draft placement by ranking.
      0 = fully random draw.
    </Form.Text>
  </Form.Group>
```

No doubles-method radio buttons for group draw — snake draft always uses pair scores for doubles (no variant needed, unlike knockout).

## Interaction Contract

### State machine transitions (matches knockout pattern)

```
SCHEDULED + registration open     → State A (Close Registration button)
SCHEDULED + registration closed
  + no groups                     → State B (Configure + Generate)
SCHEDULED + registration closed
  + groups exist                  → State C (View + Swap + Regenerate + Revert)
IN_PROGRESS + groups exist        → State D (Revert-only)
COMPLETED or CANCELLED            → null (no render)
```

### Async feedback rules (matches existing BracketGenerationSection patterns)

- All async operations: disable triggering button, show inline `Spinner as="span"` inside button text
- Generation: full card body spinner overlay during re-generation (not first generation — button spinner suffices)
- Error: `Alert variant="danger" dismissible` at top of Card.Body, cleared on next attempt
- Success: `Alert variant="success" dismissible` auto-clears after 5000ms

### Group size preview reactivity

- Renders immediately when `groupCount >= 2` and `playerCount >= 4`
- Clears when input is empty or invalid
- Inline below the group count input — not a toast, not a modal

### Swap flow

1. Organizer selects Player A from dropdown (all players across all groups, grouped by group label)
2. Organizer selects Player B from dropdown (same list, but Player A's group filtered out to force cross-group swap)
3. "Swap Players" button becomes enabled
4. On confirm: POST to server, mutateFormatStructure + mutateMatches, re-render GroupStandingsTable
5. No pending-swap queue — each swap commits immediately (unlike knockout slot editor which batches)

### Minimum interaction thresholds

- Generate Group Draw button: disabled when `playerCount < 4`
- Generate Group Draw button: disabled when `groupCount` is empty, less than 2, or exceeds `Math.floor(playerCount / 2)`
- Seeded rounds input: clamped to range [0, groupCount]; validated client-side before enabling button
- Swap button: disabled until both Player A and Player B are selected and they belong to different groups
