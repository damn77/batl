  <action>
Create `scripts/check-ci.sh` — a bash script that:

1. Accepts an optional `--branch` argument (defaults to current branch via `git branch --show-current`)
2. Uses `gh run list --repo damn77/batl --branch "$BRANCH" --limit 1 --json status,conclusion,name,event,headBranch,url` to get the latest CI run
3. If no runs found for the branch, prints "No CI runs found for branch $BRANCH" and exits 0 (not an error — push may not have triggered CI yet if branch has no PR and isn't master)
4. Parses the JSON output:
   - If status is "in_progress" or "queued": prints "CI is running... (run URL)" and exits 0
   - If conclusion is "success": prints "CI passed (run URL)" and exits 0
   - If conclusion is "failure" or "cancelled": prints "CI FAILED (run URL)" and exits 1
5. Make the script executable (`chmod +x`)

Keep it simple — no dependencies beyond `gh` and `jq` (or pure bash JSON parsing with `gh --jq`). Use `gh`'s built-in `--jq` flag to avoid requiring jq as a separate dependency.

The script should be usable standalone: `./scripts/check-ci.sh` or `./scripts/check-ci.sh --branch main`

  <action>
**Part A — PostToolUse hook in `.claude/settings.local.json`:**

Add a new entry to the existing `hooks.PostToolUse` array that triggers after Bash commands containing `git push`:

```json
{
  "matcher": "Bash",
  "hooks": [
    {
      "type": "command",
      "command": "if echo \"$command\" | grep -q 'git push'; then echo '--- CI Status Check ---'; bash scripts/check-ci.sh 2>&1; fi",
      "timeout": 30
    }
  ]
}
```

This matches any Bash tool use, checks if the command contained "git push", and if so runs the CI check script. The 30-second timeout allows for the gh API call.

Preserve ALL existing hooks (the Write and Edit eslint hooks). Add the new Bash hook entry alongside them.

**Part B — CLAUDE.md CI instruction:**

Add a new section to CLAUDE.md (after the "Git Workflow" section) titled "## CI Pipeline Checks":

```
## CI Pipeline Checks

**Before creating a pull request** (via `gh pr create` or any shipping workflow):
1. Run `bash scripts/check-ci.sh` to check the CI pipeline status
2. If CI is failing (exit code 1), DO NOT create the PR — fix the issue first
3. If CI is in progress, wait and re-check before proceeding
4. Only create the PR when CI passes or no runs exist yet

This check also runs automatically after every `git push` via a PostToolUse hook.
```

This ensures Claude checks CI regardless of whether it uses GSD workflows or direct git/gh commands.
