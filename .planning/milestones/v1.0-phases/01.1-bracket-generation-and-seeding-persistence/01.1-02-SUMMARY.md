## Decisions Made

- **jest.unstable_mockModule() instead of jest.mock()**: Jest 30 with `--experimental-vm-modules` requires `unstable_mockModule()` for ES module dependencies. The service's `new PrismaClient()` runs at module initialization, so the mock must be registered before the dynamic import via `await import(...)`. Classic `jest.mock()` calls were silently ignored, causing TOURNAMENT_NOT_FOUND errors (real Prisma returned undefined).

- **Tasks 1 and 2 committed together**: The implementation and the test update are tightly coupled — neither is meaningful without the other. A service with failing tests or tests without an implementation both represent incomplete work.

- **bracketId set on Match records**: The Match schema has both `roundId` and `bracketId` (both nullable). Setting bracketId ensures matches are discoverable via the Bracket relationship in addition to the Round relationship.

- **regenerateBracket() as explicit alias**: generateBracket() already handles deletion + recreation, but having a named `regenerateBracket()` function makes call sites self-documenting about intent.

## Deviations from Plan

None - plan executed exactly as written. The only implementation difference from the plan's suggested code pattern was discovering `jest.mock()` doesn't work with ES modules in Jest 30 (switched to `jest.unstable_mockModule()` + dynamic `await import()`) — this is the correct documented approach for Jest ES module mocking, not a deviation from the plan's intent.

## Issues Encountered

- `jest.mock('@prisma/client', ...)` silently failed with `--experimental-vm-modules` in Jest 30: all `prisma.tournament.findUnique` calls returned `undefined` instead of the mock value, causing TOURNAMENT_NOT_FOUND on every test. Root cause: Jest hoists `jest.mock()` calls but the ES module loading order with VM modules means the real PrismaClient constructor was already invoked before the mock could replace it.
- Resolution: Switched to `jest.unstable_mockModule()` (the Jest-recommended API for ES modules) combined with `await import(...)` after mock registration. All 16 tests GREEN on first successful run.
