### Task 2: Human Verification — Bugs Found and Fixed

Five issues discovered during manual testing of the 8-scenario checklist:

**Bug 1 (8cebc3b): Wrong registrations endpoint in BracketGenerationSection**
- `GET /v1/registrations?tournamentId=` does not exist; correct endpoint is `GET /tournaments/:id/registrations`
- Response shape also mismatched: fixed to handle both `reg.player` (singles) and `reg.pair` (doubles)

**Bug 2 (8cebc3b): `registrationClosed` missing from tournament GET response**
- `tournamentController` explicit field select omitted `registrationClosed`
- Frontend always received `undefined`, so State A (close registration) never transitioned to State B (generate draw)
- Fixed: added `registrationClosed` to the controller's select object

**Bug 3 (2dc7914): TournamentInfoPanel badge ignored `registrationClosed`**
- Status badge computed registration status from dates only; organizer's manual close action had no effect on the displayed badge
- Fixed: badge now checks `tournament.registrationClosed` in addition to the date comparison

**Bug 4 (5a6250d): Only 2 players appeared in generated brackets**
- Root cause 1: `generateSeededBracket()` places seeds but leaves remaining slots null; unseeded players were never distributed into empty positions. Fixed: after seeding, shuffle unseeded players deterministically (seedrandom) and fill empty non-BYE positions.
- Root cause 2: BYE detection used position-index (`pos.isBye`) instead of match-index (`structure[matchIdx]`), causing match 1 to incorrectly become a BYE. Fixed: use `structure[matchIdx]` directly.
- Tests updated to use match-indexed `byeMatchAt` parameter and corrected assertions.

**Bug 5 (3ebd8a2 + f8929d7): Missing player and wrong slot order in BYE matches**
- When seed2 had no ranking, `placeTwoSeeds` left `pos[bracketSize-1]` null; the fill loop skipped all BYE match positions, so this slot was never populated. Fixed: pre-compute exact BYE slot indices (top-half: odd index = BYE slot; bottom-half: even index = BYE slot) and fill the player slot.
- `placeTwoSeeds` places seed2 at the odd/second slot of the bottom BYE match, but UI convention is player-first. Fixed: after getting positions, swap any entity in an odd BYE slot to the even (first) slot so every BYE match displays as "Player – BYE".

### Post-verification refinements (2026-02-28)

- **Removed KnockoutBracket from BracketGenerationSection**: organizer draw workflow only needs the slot editor; the full bracket visualization is shown in the read-only (player/public) view
- **Slot editor restricted to round 1**: later rounds are empty at draw stage; only round 1 slots are editable
- **Auto-swap in slot editor**: selecting a player who is already assigned elsewhere now automatically swaps the two players rather than duplicating the name; the displaced player moves to the vacated slot
