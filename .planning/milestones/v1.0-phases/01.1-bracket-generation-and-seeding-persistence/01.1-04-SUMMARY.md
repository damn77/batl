## Decisions Made

- **Slot editor as separate section below KnockoutBracket**: KnockoutBracket fetches its own matches internally via `useMatches` and doesn't support an editMode prop. Rather than modifying the existing component, the slot editor is rendered separately below the bracket visualization. This keeps the view component unchanged and ensures backward compatibility.

- **Dual BYE detection**: `match.isBye || match.status === 'BYE'` handles both the boolean flag (new field from Phase 01.1-01) and the legacy string status format. This prevents BYE slots from showing dropdowns even if only one detection method is populated.

- **Single registeredPlayers fetch**: `useEffect` on mount loads players once and the array is reused across all three states. State A shows the count, State B shows the full list, State C populates all dropdowns. This avoids redundant API calls.

- **Guard: `mutateFormatStructure` and `mutateMatches` both called after generate**: SWR deduplication can suppress a refetch if the key hasn't changed within the dedup interval. Calling both mutates forces cache invalidation and ensures KnockoutBracket gets fresh bracket/match data immediately after generation.
