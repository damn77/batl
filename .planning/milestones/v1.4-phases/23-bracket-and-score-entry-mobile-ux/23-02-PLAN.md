**SetsScoreForm.jsx changes:**

Change ALL three Form.Control inputs (player1Score, player2Score, tiebreakScore) from `type="number"` to `type="text" inputMode="numeric" pattern="[0-9]*"`. This shows the integer-only keypad on iOS Safari (no decimal key). The `pattern="[0-9]*"` attribute is needed on iOS Safari specifically to trigger the numeric keypad consistently alongside inputMode.

Remove the `min` and `max` props from these inputs since `type="text"` does not support them. The existing `handleScoreChange` function already passes raw string values through to state, and sanitization happens in `buildSanitizedSets()` in MatchResultModal via parseInt. Add an `onKeyDown` handler to prevent non-numeric input: allow digits 0-9, Backspace, Tab, Arrow keys, Delete. This prevents letters from being typed since `type="text"` allows any character.

Specifically, create a shared handler at the top of the component:
```javascript
const handleNumericKeyDown = (e) => {
  if (!/^\d$/.test(e.key) && !['Backspace','Tab','ArrowLeft','ArrowRight','Delete','Enter'].includes(e.key)) {
    e.preventDefault();
  }
};
```

Apply `onKeyDown={handleNumericKeyDown}` to all three score inputs.

**BigTiebreakForm.jsx changes:**

Same changes: replace `type="number"` with `type="text" inputMode="numeric" pattern="[0-9]*"` on both player1Score and player2Score inputs. Remove `min` props. Add the same `handleNumericKeyDown` handler and apply to both inputs.

**MatchResultModal.jsx changes:**

In the special outcome "RETIRED" partial score inputs (the two `type="number"` Form.Control fields for player1Games and player2Games around line 442-458), apply the same change: `type="text" inputMode="numeric" pattern="[0-9]*"`. Remove `min="0"`. Add the same `handleNumericKeyDown` inline or as a local function.

Why `type="text" inputMode="numeric"` instead of `type="number"`: iOS Safari shows a keypad with a decimal key for `type="number"`, which is wrong for integer game scores. The `type="text" inputMode="numeric"` combination shows the correct integer-only keypad. This was already decided in STATE.md.
