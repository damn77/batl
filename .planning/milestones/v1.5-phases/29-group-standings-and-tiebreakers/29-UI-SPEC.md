## Interaction Contract

### 1. Standings Table — Position Cell

**Tied position:** Displays as a range string, e.g. `2-3`. Both tied rows show the same range.
- Font weight: 400 (not bold) when showing a range — visually signals unresolved state
- Font weight: 600 (bold) when showing a single resolved position

**Tiebreaker criterion badge:** Appears in the `#` column cell, below the position number, when the position was resolved by the tiebreaker chain.
- Rendered as `<Badge bg="secondary" className="d-block" style={{ fontSize: '10px' }}>H2H</Badge>`
- Badge text values: `H2H`, `Set diff`, `Game diff`, `Fewest games`, `Manual`
- Tooltip on hover/tap (via `OverlayTrigger placement="right"`) shows full label: "Resolved by: Head-to-head", "Resolved by: Set differential", etc.

**Manual position badge:** When manually resolved, badge reads `Manual` with `bg="warning" text="dark"`.
- Visible to ALL users (players and spectators) — transparency requirement from CONTEXT.md

### 2. Tie Alert Banner (Organizer only)

Displayed above the standings table rows of the tied group when the tiebreaker chain exhausts without fully separating all tied players.

```
[Alert variant="warning"]
  <strong>Tie at positions 2–3</strong> — All tiebreaker criteria exhausted.
  <Button variant="primary" size="sm">Resolve tie</Button>
```

- Alert appears inside the Accordion.Body for the affected group — not floating, not modal-level
- Only visible to `isOrganizer === true`
- Non-organizers see tied positions silently as "2-3" range — no alert

### 3. Stale Override Warning

Displayed when a match result changes after a manual override was saved for the same group. Shown to organizers only.

```
[Alert variant="warning"]
  Match result changed. Manual tie resolution for this group may be outdated.
  [Dismiss] [Re-resolve]
```

- "Dismiss" clears the warning flag without clearing the override
- "Re-resolve" opens the manual resolution modal again

### 4. Manual Resolution Modal

Triggered by "Resolve tie" button. Shows a compact modal with dropdown selectors.

**Title:** "Resolve Tie — Group [name]"
**Body:** One `<Form.Select>` per tied position showing all tied player names as options.
- Label: "Position 2" with select listing tied players
- Label: "Position 3" with select listing remaining players (removes already-assigned players from options)
- Validation: All positions must have a unique selection before "Save" is enabled

**Footer:**
- "Keep Standings" (variant="outline-secondary") — closes modal, no change
- "Save Resolution" (variant="primary") — posts override to API, closes modal

**Mobile:** Modal uses `fullscreen="sm-down"` for usability at 375px — consistent with `MatchResultModal` pattern.

### 5. Points Column Removal

**Remove:** The `Pts` column from `GroupStandingsTable`. Remove associated `stats.points` calculation from the `useMemo` standings computation.
**Remove:** The legend text `Pts = Points (2 for win)` from the table footer.
**Keep:** All other columns — `#`, Player, P, W, L, Sets W-L (hidden on mobile), S +/-, Games W-L (hidden on mobile), G +/-

**Source:** CONTEXT.md decision "Remove the 'Points' column entirely."

### 6. Loading and Error States

| State | Treatment |
|-------|-----------|
| Standings loading | `<Spinner animation="border" size="sm" />` centered in table area, `text-muted small mt-2` label "Loading standings..." |
| Standings fetch error | `<Alert variant="danger">Failed to load standings. Please try again.</Alert>` |
| No entities in group | Standings table renders with zero-stat rows — no empty state needed (group always has participants) |
| Manual resolution save error | Inline `<Alert variant="danger" className="mt-2">` inside modal body, below selects |

| Breakpoint | Column visibility | Touch targets |
|------------|------------------|---------------|
| ≥576px (sm+) | All columns visible: #, Player, P, W, L, Sets W-L, S+/-, Games W-L, G+/- | Default Bootstrap sizing |
| <576px (mobile) | Hidden: Sets W-L (`d-none d-sm-table-cell`), Games W-L (`d-none d-sm-table-cell`) | 44px min-height on match rows, "Resolve tie" button, modal selects |

Position cell with tiebreaker badge: badge uses `style={{ fontSize: '10px' }}` to fit within the narrow `#` column (width: 40px) at all breakpoints. If badge overflows, wrap in `<div style={{ width: '40px' }}>`.

Tiebreaker tooltip: touch devices tap to show, tap elsewhere to dismiss. `OverlayTrigger trigger={['hover', 'focus', 'click']}`.
