Create `backend/src/services/groupStandingsService.js` with the following exports:

**Pure functions (independently testable, no DB access):**

1. `buildEntityStats(entities, matches)` — Takes array of `{id, name}` entities and array of completed match records. Returns Map of `entityId -> { entity, played, wins, losses, setsWon, setsLost, gamesWon, gamesLost, setDiff, gameDiff, totalGames }`. Match winner determination: `result.winner === 'PLAYER1'` maps to entity whose id matches `match.player1Id || match.pair1Id`. Only process matches with `status === 'COMPLETED'` and `result` not null.

2. `sortWithTiebreakers(statsArray, override)` — Takes array of entity stats objects and optional override `{ positions: [{entityId, position}], resultSnapshotAt }`. Returns sorted array with added fields: `position` (number), `tiedRange` (string like '2-3' or null), `tiebreakerCriterion` (one of 'H2H', 'Set diff', 'Game diff', 'Fewest games', 'Manual', or null), `isManual` (boolean).

   Algorithm:
   a. Sort all entities by wins descending
   b. Group entities with equal wins into tied subsets
   c. For each tied subset with >1 entity, apply recursive tiebreaker:
      - Level 2 (H2H): Build mini-table of matches only among tied subset. Count wins within mini-table. If all entities have different win counts → resolved. If a cycle exists (use Kahn's algorithm: all entities have in-degree >= 1 among the subset) → skip H2H, fall through.
      - Level 3 (Set diff): Compare setDiff among tied subset. If different → resolved with 'Set diff'.
      - Level 4 (Game diff): Compare gameDiff among tied subset. If different → resolved with 'Game diff'.
      - Level 5 (Fewest total games): Compare totalGames ascending among tied subset. Fewer is better (rewards efficiency). If different → resolved with 'Fewest games'.
      - Level 6 (Manual): Check if override contains positions for all entities in tied subset. If yes → apply override, set isManual=true, tiebreakerCriterion='Manual'. If no → entities remain tied, tiedRange set to range string.
   d. **Partial resolution**: At each level, if the criterion separates SOME but not all entities in the subset, resolve the separated ones and recursively continue the chain for the remaining still-tied group.
   e. Assign position numbers: resolved entities get sequential position; tied entities all get the same tiedRange string (e.g., '2-3').

3. `detectH2HCycle(subset, h2hWins)` — Takes subset of entity IDs and Map of `winnerId -> Set(loserIds)` within the subset. Returns boolean (true = cycle exists). Implementation: Kahn's algorithm — compute in-degree for each entity; if all entities have in-degree >= 1, it's cyclic.

4. `computeGroupStandings(entities, matches, override)` — Orchestrator: calls buildEntityStats, then sortWithTiebreakers. Returns `{ standings: [...], unresolvedTies: [{range, entityIds}], hasManualOverride: boolean, overrideIsStale: boolean }`.

**DB-backed function:**

5. `getGroupStandings(groupId)` — Fetches group with participants (including player/pair relations), fetches completed matches for the group (with player1/player2/pair1/pair2 includes), fetches GroupTieResolution for the group, builds entity list from participants, calls computeGroupStandings, checks stale status by comparing `max(match.completedAt)` to `override.resultSnapshotAt` (use `>` strictly after, not `>=`). Returns the full response shape.
