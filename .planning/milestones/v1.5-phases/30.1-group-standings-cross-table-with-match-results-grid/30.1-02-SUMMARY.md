**Responsive tabbed GroupStandingsTable with CrossTable as default Results view, bidirectional cross-highlighting on desktop, and 3-tab mobile layout with match list as dedicated tab**

## Accomplishments

- Replaced flat standings+match-list layout with responsive Tab.Container structure
- Integrated CrossTable as the default "Results" tab on both breakpoints
- Implemented bidirectional cross-highlighting: hover match row highlights cross-table cells; hover cross-table cell highlights match row (desktop only)

## Task Commits

1. **Task 1: Restructure GroupStandingsTable with tabbed layout and CrossTable integration** - `9248b39` (feat)

## Files Created/Modified

- `frontend/src/components/GroupStandingsTable.jsx` - Restructured with Tab.Container (2-tab desktop / 3-tab mobile), CrossTable import, highlightedMatchId state, cross-highlighting handlers, removed showMatches toggle

## Decisions Made

- **crossTableEntities from match participants:** group.players and match.player1/player2 can diverge (data integrity edge case — a player removed from the group roster after matches were created). The cross-table now uses a useMemo that extracts unique entities from match participants, making the table authoritative about who actually played.
- **DB player ordering in modal accepted:** MatchResultModal shows players in database creation order, not necessarily the row-player-first order locked in CONTEXT.md. The cross-table scores are still perspective-correct (getCellScore uses rowEntity.id). User confirmed this is acceptable.
- **Desktop match list always below tabs:** Matches are context for interpreting the cross-table, not a separate primary view. Always visible below tabs on desktop without needing to switch tabs.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed cross-table showing wrong entity set when group roster diverges from match participants**
- **Found during:** Task 2 (visual verification)
- **Issue:** CrossTable received `entities` prop derived from `group.players`, but if a player was removed from the group roster after matches were created, they would disappear from the table even though their match results still exist. Conversely, empty rows/columns could appear for players with no matches.
- **Fix:** Added `crossTableEntities` useMemo in GroupStandingsTable that builds the entity list from unique participants in the matches array, ensuring the table always shows exactly who played.
- **Files modified:** `frontend/src/components/GroupStandingsTable.jsx`
- **Committed in:** `6ff9ef7`
