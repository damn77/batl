#### 1. Cross-table visual rendering

**Test:** Navigate to a GROUP or COMBINED tournament with a group that has completed and incomplete matches. View the group section.
**Expected:** "Results" tab is active by default showing a grid. Column headers display player initials (e.g. "AL" for Alice Liddell, "BM" for Bob Martin). Full names appear in row labels. Completed cells show scores in bold green (win) or grey (loss). Diagonal cells are grey. Unplayed cells show an en-dash character.
**Why human:** CSS color rendering, initials accuracy, and visual layout cannot be verified by static analysis.

#### 2. Cell click opens MatchResultModal

**Test:** Click a completed cell (score displayed) and an unplayed cell (en-dash).
**Expected:** A modal opens showing the match. For an unplayed cell where no match exists yet (e.g. unscheduled), no modal opens (guard in `handleCellClick` prevents it). For a scheduled match cell, the modal should open for result entry.
**Why human:** Modal open/close behavior requires interactive testing.

#### 3. Desktop 2-tab / mobile 3-tab responsive layout

**Test:** View the group section at 768px+ viewport width and at 375px.
**Expected:** At 768px+: two tabs visible (Results, Standings); match list always visible below the tab content. At 375px: three tabs (Results, Standings, Matches); match list only visible when "Matches" tab is active.
**Why human:** Responsive CSS layout (Bootstrap breakpoints) requires viewport resize testing.

#### 4. Bidirectional cross-highlighting (desktop)

**Test:** At desktop viewport, hover a row in the match list below the tabs. Then hover a cross-table cell.
**Expected:** Hovering a match row turns the corresponding cross-table cell(s) light blue. Hovering a cross-table cell turns the corresponding match row light blue. Switching tabs clears any active highlight.
**Why human:** Hover state and visual color effect require interactive observation.

#### 5. Single-set / BIG_TIEBREAK score format

**Test:** Open a tournament using BIG_TIEBREAK or winningSets=1 scoring. View a completed match cell.
**Expected:** Cell shows "6-4" (game score) not "1-0" (meaningless set count).
**Why human:** Requires a tournament with this scoring format to be available in test data.

#### 6. SWR mutation updates both symmetric cells after result entry

**Test:** Click an unplayed cell for a match with two known players. Enter a result and submit. Observe both cells [A,B] and [B,A].
**Expected:** Both cells immediately update to show the score from each player's perspective (e.g. one shows "6-4", the other shows "4-6") without page refresh.
