Create `backend/src/services/groupPersistenceService.js` with these exports:

1. **`makeError(code, message)`** — same pattern as bracketPersistenceService

2. **`snakeDraft(rankedEntities, groupCount, seededRounds)`** — pure function
   - Takes array of entities sorted by ranking (best first), groupCount, seededRounds
   - Returns array of groupCount arrays, each containing entities for that group
   - Round 1: fill groups 0→(groupCount-1) (forward)
   - Round 2: fill groups (groupCount-1)→0 (reverse)
   - Alternates direction each round
   - Only processes `seededRounds * groupCount` entities from the ranked list
   - Returns the groups (partially filled) — unseeded fill happens separately

3. **`fillUnseeded(groups, unseededEntities, randomSeed)`** — pure function
   - Shuffles unseededEntities deterministically using `shuffle(array, seed)` from seedingPlacementService
   - Snake-fills remaining slots across groups (continuing the snake direction from where seeded rounds left off)
   - First groups get extra players when uneven (natural snake fill order)

4. **`generateCircleRoundRobin(participants, groupNumber)`** — pure function
   - Implements standard circle method for round-robin scheduling
   - For N participants: if N is odd, add null BYE sentinel to make even
   - Generate N-1 rounds (where N is the even-padded count)
   - Each round: pair position[i] with position[N-1-i] for i in 0..N/2-1
   - After each round: fix position[0], rotate positions[1..N-1] (pop last, insert at index 1)
   - Skip fixtures where either participant is null (BYE)
   - Return array of `{ round, p1, p2 }` objects
   - Match numbering: `groupNumber * 100 + matchIndex + 1` (Group 1: 101+, Group 2: 201+)

5. **`validateGroupBalance(playerCount, groupCount)`** — pure function
   - Returns `{ valid, sizes, error }`
   - Valid if: `Math.ceil(playerCount/groupCount) - Math.floor(playerCount/groupCount) <= 1`
   - Also valid only if minimum group has >= 2 players
   - `sizes`: string preview like "2 of 4, 2 of 3"

6. **`generateGroupDraw(tournamentId, options)`** — async, main export
   - `options`: `{ groupCount, seededRounds = 0, randomSeed }`
   - Guards: tournament must exist, status SCHEDULED, registrationClosed true, no existing groups (or delete them)
   - Load entities: branch on `category.type === 'DOUBLES'` (use pairRegistration for doubles, tournamentRegistration for singles — same pattern as bracketPersistenceService lines 147-160)
   - Validate: playerCount >= 4, groupCount >= 2, groupCount <= Math.floor(playerCount/2)
   - Validate balance: `validateGroupBalance(playerCount, groupCount)`
   - If seededRounds > 0: call `getSeededPlayers(categoryId, seededRounds * groupCount)` to get ranked entities
   - Run snakeDraft for seeded rounds
   - Run fillUnseeded for remaining
   - Generate randomSeed if not provided: `seedrandom().quick().toString(36).substring(2)` or use `createRandomSeed()` from seedingPlacementService
   - Atomic `prisma.$transaction`:
     a. Delete existing: `tx.match.deleteMany({ where: { groupId: { in: existingGroupIds } } })` then `tx.groupParticipant.deleteMany(...)` then `tx.group.deleteMany({ where: { tournamentId } })`
     b. Create Groups (1 per groupCount): `tx.group.create({ data: { tournamentId, groupNumber: i+1, groupSize: groups[i].length } })`
     c. Create GroupParticipants per group: for singles use `playerId`, for doubles use `pairId`. Set `seedPosition` for seeded entities (1-indexed within tournament, not within group)
     d. Create Rounds per group: `tx.round.create({ data: { tournamentId, bracketId: null, roundNumber } })` — one round record per round-robin round, shared across groups at same round number. Actually each group has independent rounds — create Round per group per roundNumber with bracketId null.
     e. Generate circle round-robin per group → create Matches with `{ tournamentId, groupId, roundId, matchNumber, player1Id/pair1Id, player2Id/pair2Id, status: 'SCHEDULED' }`
   - Return `{ groups: [...], participantCount, matchCount, randomSeed }`
