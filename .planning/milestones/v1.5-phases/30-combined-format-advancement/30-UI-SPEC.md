## Spacing Scale

Declared values (must be multiples of 4, standard set: 4, 8, 16, 24, 32, 48, 64):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xs: 0.25rem`) | Icon gaps, inline badge padding |
| sm | 8px (`--spacing-sm: 0.5rem`) | Compact element spacing, tight card body |
| lg | 16px (`--spacing-lg: 1rem`) | Default element spacing, container gaps |
| xl | 24px (`--spacing-xl: 1.5rem`) | Major section breaks |

Exceptions:

| Token | Value | Justification |
|-------|-------|---------------|
| `--spacing-md` | 12px (0.75rem) | Pre-existing project token from `app.css` used in card body padding; 12px is a multiple of 4 but outside the standard set. Retained for codebase consistency — do not redefine. |
| Touch targets (mobile) | min-height 44px | Global rule in `app.css` for `<= 575.98px` — all interactive elements |
| Group-to-knockout gap | 48px (`mb-5`, Bootstrap 3rem) | Preserves existing visual separation between group stage accordion and knockout section in `CombinedFormatDisplay.jsx` |
| Bracket card vertical gap | 24px (`gap-4`, Bootstrap 1.5rem) | Preserves existing stacking gap between multiple brackets in `CombinedFormatDisplay.jsx` |

## Typography

**Source:** `app.css` `:root` tokens applied globally. Do not deviate.

**Note on scale gaps:** Body (15px) and section heading (17px) are 2px apart. This tight spacing is intentional — inherited from project tokens in `app.css`, not introduced by this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px (`--font-size-base: 0.9375rem`) | 400 (regular) | 1.5 (Bootstrap default) |
| Label / Small | 13px (`--font-size-sm: 0.8125rem`) | 400 (regular) | 1.5 |
| Section heading | 17px (`--font-size-lg: 1.0625rem`) via `h6` | 600 (semibold, `fw-bold`) | 1.2 |
| Display / Page heading | 20px (`h5` Bootstrap default) | 600 (semibold, `fw-bold`) | 1.2 |

Mobile overrides (auto-applied by `app.css` at `<= 575.98px`):
- `h1` → 24px, `h2` → 20px, `h3` → 17.6px

## Color

**Source:** Bootstrap 5.3 semantic palette + `app.css` tokens. No custom hex values — use Bootstrap variant names.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8f9fa` (`--color-surface`, Bootstrap `bg-light`) | Page background, table headers (`table-light`), card surfaces |
| Secondary (30%) | `#ffffff` (Bootstrap white) | Card bodies, modal backgrounds, accordion bodies |
| Accent (10%) | Bootstrap `primary` (blue) | Primary CTAs only — see reserved list below |
| Destructive | Bootstrap `danger` (red) | Revert advancement action only |

**Accent (`primary`) reserved for:**
1. "Confirm & Generate Brackets" button — the single commit action
2. "Generate Knockout Bracket" button in group completion banner (existing pattern in `CombinedFormatDisplay`)
3. "Resolve tie" / "Re-resolve" buttons (existing pattern in `GroupStandingsTable`)
4. "Save Resolution" in manual tie modal (existing pattern)

**Semantic alert colors (Bootstrap Alert variants):**
- `success` — Group stage complete banner (existing), advancement confirmed state
- `warning` — Unresolved ties blocking advancement, stale override warning
- `danger` — Error states, revert confirmation
- `info` — Advancement preview informational callout, secondary bracket optional note

**Never use accent/primary for:** outline buttons, secondary actions, dismiss buttons (use `outline-secondary`), badges (use `secondary` or `primary` bg based on context).

## Interaction States

### Advancement Configuration (at tournament creation — `CombinedConfigPanel`)

| State | Visual |
|-------|--------|
| Default | Two new number inputs below existing group config: "Players advancing to main bracket (Top N)" and "Players advancing to secondary bracket (Next M, optional)" |
| Disabled (tournament started) | Both inputs `disabled` — grey out with Bootstrap disabled state |
| Validation success | Bootstrap `Alert variant="success"` showing waterfall summary: "{playerCount} players → {groups} groups → {N} main + {M} secondary + {R} eliminated" |
| Validation warning (invalid count) | Bootstrap `Alert variant="warning"` explaining why — byes are allowed, so only block on M+N > group total |

### Group Completion Banner (existing `CombinedFormatDisplay` line 55 — rewire)

| State | Visual |
|-------|--------|
| All groups complete, no brackets, unresolved ties | `Alert variant="warning"` — "Resolve all tied standings before advancing. {N} group(s) have unresolved ties." Button: "Generate Knockout Bracket" — disabled |
| All groups complete, no brackets, ties resolved | `Alert variant="success"` — existing copy "Group stage complete — Ready to generate knockout bracket". Button: "Generate Knockout Bracket" — enabled, `variant="primary"` |
| Loading preview | Button shows `Spinner` inline, disabled |
| Error fetching preview | `Alert variant="danger"` — error message inline, button re-enabled |

### Advancement Preview Modal (`AdvancementPreviewModal`)

| State | Visual |
|-------|--------|
| Loading | Modal open, `Spinner` centered, no content yet |
| Preview loaded | Waterfall table: columns "Player/Pair", "Group", "Position", "Advances To". Rows grouped by bracket label. Spillover rows (cross-group picks) shown with `Badge bg="info"` text "Spillover". Bracket summary footer: "Main Bracket: {N} players ({B} byes)" and "Secondary Bracket: {M} players ({B} byes)" if applicable. Eliminated players shown at bottom in muted text |
| Confirm action | "Confirm & Generate Brackets" button `variant="primary"` — full width on mobile |
| Generating (loading) | Button shows `Spinner` inline, disabled, text "Generating..." |
| Success | Modal closes, page reloads with brackets visible |
| Error | `Alert variant="danger"` inside modal footer — problem + "Please try again" |

### Post-Advancement Layout (`CombinedFormatDisplay`)

| State | Visual |
|-------|--------|
| Brackets exist | Groups section: Accordion `defaultActiveKey={null}` (collapsed by default, read-only). Knockout section: brackets displayed prominently. Accordion header shows lock icon text "(Read-only)" via `Badge bg="secondary"` |
| Main bracket only | Single `KnockoutBracket` labeled "Main Bracket" (bracket name field) |
| Main + secondary | Two `KnockoutBracket` components stacked with `gap-4`. Labels: "Main Bracket" and "Secondary Bracket" |
| Revert available (no knockout results) | `Alert variant="warning"` below brackets: "Undo advancement to edit group results." Button: "Revert Advancement" `variant="outline-danger"` |

### Advancement Revert

| State | Visual |
|-------|--------|
| Revert button clicked | Confirmation `Modal` — title "Revert Advancement", body "This will delete all knockout brackets and unlock group results for editing." Footer: "Keep Brackets" `variant="outline-secondary"` + "Revert Advancement" `variant="danger"` |
| Reverting | "Revert Advancement" button shows `Spinner`, disabled |
| Revert success | Modal closes, page returns to group-complete banner state |
| Revert error | `Alert variant="danger"` inside modal footer |
