**Current problems to fix:**
1. `const points = 1000 - i * 100;` -- produces 1000, 900, 800, 700... (linear ladder)
2. `tournamentCount: 5` -- hardcoded same value for every player
3. `seedingScore: points` -- identical to totalPoints (unrealistic; seedingScore = sum of best N results, which is always less than totalPoints)

**Implementation approach:**

Replace the generic formula with a predefined array of realistic ranking profiles. Each profile has `{ totalPoints, tournamentCount, seedingScore }` with realistic values:


Place this array near the top of the `seedRanking()` function or just above it.

**Modify the loop inside `seedRanking()`:**

Replace:
```javascript
const points = 1000 - i * 100;
// ...
totalPoints: points,
tournamentCount: 5,
seedingScore: points
```

With:
```javascript
const profile = RANKING_PROFILES[i % RANKING_PROFILES.length];
// ...
totalPoints: profile.totalPoints,
tournamentCount: profile.tournamentCount,
seedingScore: profile.seedingScore
```

This ensures:
- The `i % RANKING_PROFILES.length` wrapping handles any number of entities (currently max is 8 for Men's Open SINGLES)
- Rankings are still ordered by decreasing points (RANKING_PROFILES is pre-sorted descending by totalPoints)
- But the gaps between ranks are irregular (not uniform 100-point gaps)
- Tournament counts vary between 3 and 8
- Seeding scores are always less than totalPoints (realistic: best 7 of N tournaments)

**Do NOT modify:**
- The Mixed Doubles Open rankings (PAIR/MEN/WOMEN) which correctly use 0 points (TOURN-02)
- The men's/women's doubles pair seedingScores (1900, 1500, 1100, 700, 1720, 1400) which are intentionally set
- The structure of the `seedRanking()` function (upsert pattern, ranking creation, entry creation)
- Any other section of seed.js

**Important:** Keep the rank field (`rank: i + 1`) as-is. The rank is determined by position, and since RANKING_PROFILES is sorted descending, the ranking order remains correct.
