## Decisions Made

- **pair1Id ?? player1Id pattern:** Doubles matches have pair1Id set and player1Id null; singles have the reverse. Nullish coalescing gives the correct entity ID for both tournament types without a branching if/else.
- **advanceBracketSlot fallback:** When resultJson.winner is absent (consolationOptOutService call site where no result has been written yet), derive slot direction by comparing winnerId against updatedMatch.pair1Id/player1Id directly. This is unambiguous because the opponentId always equals either pair1Id or pair2Id.
