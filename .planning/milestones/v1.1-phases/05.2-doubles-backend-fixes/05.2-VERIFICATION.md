### Gaps Summary

No gaps. Both targeted line-level patches are applied exactly as specified in the plan:

- **BREAK 1** (matchResultService.js): The single-line `winnerId` derivation was expanded to the `pair1Id ?? player1Id` / `pair2Id ?? player2Id` pattern. For any doubles match where `pair1Id` is set and `player1Id` is null, `winnerId` is now the correct pair ID rather than null.

- **BREAK 2** (tournamentLifecycleService.js): The `advanceBracketSlot` function now has a two-branch `winnerIsPlayer1` derivation — primary branch uses `resultJson.winner` when present; fallback branch uses positional comparison (`pair1Id === winnerId || player1Id === winnerId`) when `resultJson` is null or `winner` field is absent. This directly fixes the consolation opt-out path where no match result exists at call time.

Both commits (`1f3c672`, `aa236d6`) are verified in git history. All 166 backend tests pass with zero regressions.
