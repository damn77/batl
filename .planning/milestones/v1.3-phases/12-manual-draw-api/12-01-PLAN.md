    Modify `generateBracket(tournamentId, options = {})`:

    1. Extract `mode` from options: `const { randomSeed, mode = 'seeded' } = options;`

    2. **Manual mode branch** — after Step 6 (player count guard), if `mode === 'manual'`:
       - Skip Step 7 entirely (no call to `generateSeededBracket`)

       - Call `getBracketByPlayerCount(playerCount)` to get `{ bracketSize, structure }`
       - Parse the structure string (remove spaces) for BYE detection
       - Continue to Step 8 (persist) but with ALL non-BYE positions having null entityId

    3. **In the Step 8 transaction** (Prisma `$transaction`):
       - When creating the Bracket record, add `drawMode: mode === 'manual' ? 'MANUAL' : 'SEEDED'`
       - For manual mode, Round 1 match creation:
         - BYE matches: `isBye: true`, `status: 'BYE'`, all player/pair fields null (no player placed yet)
         - Non-BYE matches: `isBye: false`, `status: 'SCHEDULED'`, all player/pair fields null
       - For manual mode, skip Step 7b (unseeded fill) and Step 7c (entity ID validation)
       - For manual mode, skip Step 8d (BYE Round 2 pre-population) — no players to advance yet
       - Rounds 2+ are still placeholders with null slots (same as seeded mode)

    4. **Seeded mode** (existing behavior): Unchanged. Add `drawMode: 'SEEDED'` to the Bracket create call.

    5. **Consolation bracket** (Step 8e): Generate for both modes when `matchGuarantee === 'MATCH_2'`. The consolation bracket itself is always empty at generation time (populated during tournament play). No change needed.

    6. Return result includes `bracket.drawMode` so the controller can include it in the response.

    **Important implementation details:**
    - Import `getBracketByPlayerCount` from `'./bracketService.js'` at the top of the file (alongside existing imports)
    - The manual mode branch should be clearly separated with a comment block: `// --- MANUAL MODE: skip seeding, create empty bracket ---`
    - BYE positions in manual mode are determined by the bracket template structure string, same as seeded mode
    - The structure string format: '0' = preliminary match (non-BYE), '1' = BYE match. Remove spaces before use.
    - Manual mode minimum player count remains 4 (same guard as seeded)

    2. **Load all registered players/pairs for the tournament:**
       - For DOUBLES (`category.type === 'DOUBLES'`): query `prisma.pairRegistration.findMany({ where: { tournamentId, status: 'REGISTERED' }, select: { pairId: true } })` to get all registered pair IDs
       - For SINGLES: query `prisma.tournamentRegistration.findMany({ where: { tournamentId, status: 'REGISTERED' }, select: { playerId: true } })` to get all registered player IDs

    3. **Load all Round 1 non-BYE matches in the MAIN bracket:**
       - Find the MAIN bracket: `prisma.bracket.findFirst({ where: { tournamentId, bracketType: 'MAIN' } })`
       - If no bracket exists, throw `throwError(400, 'NO_BRACKET', 'No bracket has been generated for this tournament')`
       - Find Round 1: `prisma.round.findFirst({ where: { bracketId: mainBracket.id, roundNumber: 1 } })`
       - Get all non-BYE Round 1 matches: `prisma.match.findMany({ where: { roundId: round1.id, isBye: false }, select: { player1Id, player2Id, pair1Id, pair2Id } })`

    4. **Integrity checks:**

       a. **Every non-BYE Round 1 slot must be filled:**
          - For SINGLES: check that every match has both `player1Id` and `player2Id` non-null
          - For DOUBLES: check that every match has both `pair1Id` and `pair2Id` non-null
          - Collect empty slot count

       b. **All registered players/pairs must appear somewhere in the bracket:**
          - Build a Set of all entity IDs found in Round 1 matches (including BYE matches)
          - To include BYE matches too, load ALL Round 1 matches (not just non-BYE) for this check
          - For SINGLES: collect player1Id and player2Id from all Round 1 matches
          - For DOUBLES: collect pair1Id and pair2Id from all Round 1 matches
          - Compare against the registered set to find unplaced entities

       c. **No entity appears in multiple positions:**
          - Check for duplicate IDs in the collected set
          - If duplicates found, include them in error details

    5. **If any check fails**, throw:
       ```javascript
       const err = new Error('Bracket is incomplete — not all players/pairs are placed');
       err.statusCode = 400;
       err.code = 'INCOMPLETE_BRACKET';
       err.details = {
         unplacedPlayers: unplacedList.map(id => ({ id })),
       ```

       ```javascript
       return res.status(err.statusCode).json({
       ```

       This is in `backend/src/api/tournamentController.js` in the `startTournament` function's catch block.
