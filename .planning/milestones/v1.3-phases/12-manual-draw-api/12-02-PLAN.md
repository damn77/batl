  <files>backend/src/services/bracketPersistenceService.js</files>
  <action>
    Add a new exported function `assignPosition(tournamentId, assignment)` to `bracketPersistenceService.js`.

    **Function signature:**
    ```javascript
    export async function assignPosition(tournamentId, assignment)
    ```

    **Input `assignment` object:**
    ```javascript
    {
      matchId: string,        // Target Round 1 match
      slot: 'player1' | 'player2',  // Which slot in the match
      playerId: string | null, // For singles (null to clear)
      pairId: string | null    // For doubles (null to clear)
    }
    ```

    **Implementation steps (all within a single `prisma.$transaction`):**

    **Step 1: Load and validate tournament**
    - Load tournament with category type: `prisma.tournament.findUnique({ where: { id: tournamentId }, select: { id, status, categoryId, category: { select: { type } } } })`
    - Guard: tournament must exist → `makeError('TOURNAMENT_NOT_FOUND', ...)`
    - Guard: tournament must not be IN_PROGRESS or COMPLETED → `makeError('BRACKET_LOCKED', 'Cannot modify bracket when tournament is in progress or completed')`

    **Step 2: Load the target match**
    - `prisma.match.findUnique({ where: { id: matchId }, include: { round: { select: { roundNumber, bracketId } } } })`
    - Guard: match must exist and belong to this tournament → `makeError('MATCH_NOT_FOUND', ...)`
    - Guard: match must be in Round 1 → `makeError('NOT_ROUND_1', 'Can only assign positions in Round 1')`
    - Guard: match must not be a BYE match → `makeError('BYE_SLOT_NOT_ASSIGNABLE', 'Cannot assign players to BYE match slots')`

    **Step 3: Determine entity ID**
    - `const isDoubles = tournament.category.type === 'DOUBLES';`
    - `const entityId = isDoubles ? assignment.pairId : assignment.playerId;`
    - `const entityField1 = isDoubles ? 'pair1Id' : 'player1Id';`
    - `const entityField2 = isDoubles ? 'pair2Id' : 'player2Id';`
    - `const slotField = assignment.slot === 'player1' ? entityField1 : entityField2;`

    **Step 4: If assigning (entityId is not null), validate registration and uniqueness**
    - **Registration check:** For singles, query `prisma.tournamentRegistration.findFirst({ where: { tournamentId, playerId: entityId, status: 'REGISTERED' } })`. For doubles, query `prisma.pairRegistration.findFirst({ where: { tournamentId, pairId: entityId, status: 'REGISTERED' } })`. If not found → `makeError('NOT_REGISTERED', 'Player/pair is not registered for this tournament')`
    - **Already-placed check:** Load ALL Round 1 matches in the MAIN bracket. Find the bracket via the match's `round.bracketId`. Get Round 1 (`roundNumber: 1`), then all matches in Round 1. Scan all matches for `entityId` in either player/pair slot. If found in a different match or different slot (not the exact same matchId+slot we're assigning to) → `makeError('ALREADY_PLACED', 'Player/pair is already placed at a different bracket position')`. If found in the SAME slot → return early (no-op, already assigned there).

    **Step 5: Handle current occupant of the target slot (reassignment)**
    - Read current occupant: `const currentOccupant = targetMatch[slotField];`
    - If `currentOccupant` is not null AND `currentOccupant !== entityId`:
      - **Undo Round 2 pre-population for the old occupant** if this match is BYE-adjacent:
        - Load all Round 1 matches in order (by matchNumber) to determine `posInRound` (0-indexed position of this match in Round 1)
        - Determine the "partner match" index: if this match is at even posInRound, the partner is posInRound+1; if odd, partner is posInRound-1
        - Check if the partner match is a BYE match (`isBye: true`)
        - If partner is BYE: the current slot's player was auto-advanced to Round 2. Undo it:
          - `round2Idx = Math.floor(posInRound / 2)`
          - `isPlayer1Slot = posInRound % 2 === 0`
          - Load Round 2 matches in order, find the match at `round2Idx`
          - Clear the appropriate slot: `{ [isPlayer1Slot ? entityField1 : entityField2]: null }`

    **Step 6: Clear the target slot first (if occupied)**
    - If `currentOccupant` is not null: update match to set `slotField` to null

    **Step 7: If clearing (entityId is null), also undo Round 2 pre-population**
    - Same BYE-adjacent check as Step 5 — if the cleared position was next to a BYE, remove the player from Round 2

    **Step 8: Assign the new entity (if entityId is not null)**
    - Update the target match: `{ [slotField]: entityId }`
    - **BYE auto-advance:** After assigning, check if this match is BYE-adjacent:
      - Load partner match (same as Step 5 logic)
      - If partner is BYE: auto-advance the newly assigned player to Round 2:
        - Same Round 2 index calculation as Step 5
        - Update Round 2 match: `{ [isPlayer1Slot ? entityField1 : entityField2]: entityId }`

    **Step 9: Return result**
    ```javascript
    return {
      matchId: assignment.matchId,
      slot: assignment.slot,
      entityId: entityId,
      action: entityId ? 'assigned' : 'cleared'
    };
    ```

    **Key implementation details:**
    - The entire operation MUST be in a `prisma.$transaction(async (tx) => { ... })` for atomicity
    - Use `tx` (not `prisma`) for ALL queries inside the transaction
    - The BYE-adjacent check pattern: a match is "BYE-adjacent" if it is a non-BYE match whose Round 1 partner (the match it feeds into the same Round 2 match) is a BYE match
    - Position in round (posInRound) must be calculated by loading all Round 1 matches sorted by matchNumber and finding the index
    - For BYE matches in manual mode, the BYE match itself has no player assigned (both slots null) — the BYE is structural only. When a player is assigned to the non-BYE partner match, they auto-advance to Round 2 because the BYE opponent "advances" them.
    - The `slotField` variable maps 'player1' → 'player1Id'/'pair1Id' and 'player2' → 'player2Id'/'pair2Id' depending on category type
