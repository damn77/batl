# Requirements: BATL

**Defined:** 2026-03-15
**Core Value:** A complete tournament runs from registration to final standings without the organizer touching a spreadsheet or a WhatsApp group

## v1.5 Requirements

Requirements for Group & Combined Tournaments milestone. Each maps to roadmap phases.

### Group Formation

- [ ] **GFORM-01**: Organizer can configure the number of groups and group sizes at registration close
- [ ] **GFORM-02**: System validates that group sizes are balanced (no group more than 1 player smaller than the largest)
- [ ] **GFORM-03**: Organizer can configure the number of seeded rounds for snake draft placement
- [ ] **GFORM-04**: System distributes seeded players across groups using snake draft based on ranking (one per tier per group, random within tier)
- [ ] **GFORM-05**: System randomly distributes remaining unseeded players across groups after seeded rounds
- [ ] **GFORM-06**: Organizer can manually override player-to-group assignments after automated formation
- [ ] **GFORM-07**: Group formation works for both singles (players) and doubles (pairs)

### Group Stage Play

- [ ] **GPLAY-01**: System generates round-robin match schedule within each group (every participant plays every other participant once)
- [ ] **GPLAY-02**: Player can submit match results for group stage matches using existing score entry patterns
- [ ] **GPLAY-03**: Organizer can submit and correct group stage match results
- [ ] **GPLAY-04**: Group stage matches follow the tournament's format rules (sets, tiebreaks, advantage settings)
- [ ] **GPLAY-05**: Group stage play works for both singles and doubles tournaments

### Group Standings & Tiebreakers

- [ ] **GSTAND-01**: System calculates group standings based on win/loss record
- [ ] **GSTAND-02**: System applies tiebreaker chain: set/point differential → fewer games/points played → head-to-head result
- [ ] **GSTAND-03**: System detects three-way circular tiebreaker locks (A>B, B>C, C>A) and flags them for manual resolution
- [ ] **GSTAND-04**: Organizer can manually resolve three-way tiebreaker locks by setting final positions
- [ ] **GSTAND-05**: Group standings display shows current positions, W/L record, differential, and head-to-head indicators
- [ ] **GSTAND-06**: Group standings work correctly for both singles and doubles tournaments

### Combined Format

- [ ] **COMB-01**: Organizer can configure a combined tournament (group stage → one or more knockout brackets)
- [ ] **COMB-02**: Organizer can configure which group positions advance to which knockout bracket (e.g., 1st/2nd → main, 3rd → consolation bracket)
- [ ] **COMB-03**: System supports "best of N" spillover advancement for odd-group situations (e.g., best 2 of 3 third-placed players advance)
- [ ] **COMB-04**: Spillover advancement uses cross-group ranking (win rate, differential) to select best remaining players
- [ ] **COMB-05**: System validates that advancement counts produce valid knockout bracket sizes (4-128, per Feature 009)
- [ ] **COMB-06**: Knockout brackets are generated from group results using existing bracket generation and seeding infrastructure
- [ ] **COMB-07**: Combined tournament lifecycle transitions from group phase to knockout phase when all group matches are complete
- [ ] **COMB-08**: Combined tournament completes only when all knockout brackets are finished
- [ ] **COMB-09**: Combined format works for both singles and doubles tournaments

### Advancement Configuration

- [ ] **ADV-01**: Organizer can configure advancement rules via a dedicated configuration UI
- [ ] **ADV-02**: Advancement UI shows how many players advance from each position and to which knockout bracket
- [ ] **ADV-03**: Advancement UI validates total advancement count against valid bracket sizes
- [ ] **ADV-04**: Advancement configuration can be edited before groups start but not after

### Points Integration

- [ ] **PTS-01**: Non-advancing players receive ranking points based on their final group placement
- [ ] **PTS-02**: Advancing players receive ranking points based on their knockout bracket result (superseding group placement points)
- [ ] **PTS-03**: Group-only tournaments (no knockout) award points based on final group placement
- [ ] **PTS-04**: Points calculation works correctly for both singles and doubles tournaments

### Group Visualization

- [ ] **GVIEW-01**: Tournament page displays group standings tables for GROUP and COMBINED format tournaments
- [ ] **GVIEW-02**: Group standings table shows match results, W/L record, and current positions within each group
- [ ] **GVIEW-03**: Combined tournament page shows both group standings and knockout bracket(s) based on tournament phase
- [ ] **GVIEW-04**: Group visualization is responsive and usable on mobile devices (375px viewport)
- [ ] **GVIEW-05**: Advancement status is visible on group standings (which players have advanced to knockout)

## Future Requirements

### Deferred to v1.6+

- **SWISS-01**: Swiss system pairing and result tracking
- **GUAR-01**: MATCH_1 / UNTIL_PLACEMENT guarantee levels for group tournaments
- **GVIEW-D01**: Group match schedule calendar view with time slots
- **COMB-D01**: Consolation brackets within combined format tournaments
- **GFORM-D01**: Auto-suggested group count based on player count and desired group size

## Out of Scope

| Feature | Reason |
|---------|--------|
| Swiss system tournaments | Explicitly deferred to future milestone per user decision |
| Consolation brackets in combined format | Not needed — groups → knockout tiers serve same purpose |
| Real-time live group standings updates | Polling/WebSocket adds complexity; SWR revalidateOnFocus sufficient |
| Group match time scheduling | Out of scope — matches are unscheduled, just listed for completion |
| Automated group count suggestion | Nice-to-have; organizer decides group count manually |
